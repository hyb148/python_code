                                                                                    Ah, and one more thing. Here, we only have one dataset. But in real life, you usually have something like "training data" and "new data" (or "test data"). And you have to consider that you have to use the same parameters (standard deviation) for transforming the new data that you used to transform your training data. This is why I like to use the StandardScaler from scikit-learn since it stores the parameter for your convenience.

                                                                                    This would look like this:

                                                                                    std_scale = preprocessing.StandardScaler().fit(X_train)
                                                                                    X_train_std = std_scale.transform(X_train)
                                                                                    X_new_std = std_scale.transform(X_new)

                                                                                    sklearn_pca = PCA(n_components=2)
                                                                                    X_transf = sklearn_pca.fit(X_train_std)
                                                                                    X_transf_train = sklearn_pca.transform(X_train_std)
                                                                                    X_transf_new = sklearn_pca.transform(X_new_std)

                                                                                                                Sebastian Raschka Mod MichaelJackson1234 • 6 months ago

                                                                                                                Hey, Michael,

                                                                                                                The matrix W is basically just the collection of your principal components. So, you can get them via sklearn via:

                                                                                                                # 1st set how many components you want, e.g., 4
                                                                                                                sklearn_pca = PCA(n_components=4)
                                                                                                                X_transf = sklearn_pca.fit_transform(X

                                                                                                                # get all 4 components (i.e., matrix W)
                                                                                                                sklearn_pca.components_ # all 4 components

                                                                                                                If you later decide that you only want the first 2 components, you can do e.g.,
                                                                                                                sklearn_pca.components_[:, :2] # first 2 components

                                                                                                                >Or maybe it is just easier to use matplotlib.mlab.PCA if I always want to use the correlation matrix, because it always use the correlation matrix, right?

                                                                                                                No, matplotlib.mlab is also doing a Singular Value Decomposition - for performance reasons (and maybe precision). It basically saves you the steps to create a covariance matrix (or correlation matrix), and eigendecomposition does only work on square matrices. Roughly speaking, it is more of a "direct" approach or "shortcut". In this tutorial I went with the more "classic" approach doing PCA because I find it more easier to interpret. However, I probably should add some section about SVD in future.
